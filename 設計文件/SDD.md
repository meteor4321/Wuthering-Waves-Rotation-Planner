# 鳴潮（Wuthering Waves）排軸輔助工具 - 系統設計文件

## 專案總覽（Project Overview）
* **專案定位：** 專為遊戲《鳴潮》設計的三人輪切戰鬥排軸輔助工具。
* **目標受眾：** 暫定個人自用，未來視情況開源或公開。
* **開發架構：** 純前端單頁應用程式（Single-Page Application，SPA）。
    * *說明：* 所有運算與資料儲存皆在使用者端（瀏覽器）完成，無須後端伺服器，達成零成本營運，且使用者透過網址即可跨平台直接使用。

---

## 核心機制與視覺設計（Core Mechanics & UI）

### 1. 區塊 (Block)
* 在鳴潮中, 角色的基礎招式包含:普攻A/重擊Z/共鳴技能E/共鳴解放R/聲骸技能Q/閃避D/變奏。
* **定義**：區塊為本專案中的核心機制，代表角色一個或多個基礎招式的組合。其顯示文字（label） 可由系統預設，亦可由使用者自由輸入、編輯任意文字或連招 （例如：「3AE」表示「3下普攻接一個小技能」），甚至允許輸入自訂亂碼。
* **基礎區塊**是指僅有單一基礎招式的區塊; **連招區塊**是指多個基礎招式組成的區塊
* **區塊寬度（Width）：** 由'label'文字的字元數自動撐開，不代表實際耗時。

### 2. 輸出軸 (Rotation)
* 總共有三位角色, 因此有三條輸出軸(形似泳道圖)
* 每條軸內包含許多 blocks
* 輸出軸左側顯示對應角色的名稱

### 3. 時間軸邏輯（Rotation Timeline Logic）
* **順序至上原則：** 放棄精確的秒數對齊與真實時間比例。時間軸僅呈現招式施放的「先後順序」。
    * 這裡的"時間軸"只是一個概念, 並不會實際顯示在GUI中。
* **單一執行緒（Single Thread）架構：** 由於《鳴潮》為單人站場機制，三名角色的時間軸在底層資料結構上為一個「一維陣列」。
    * 系統依照陣列順序，將不同角色的區塊繪製對應的主畫面三條泳道（Swimlane）中。
    * $Index(Block_{n}) = Index(Block_{n-1}) + 1$
* **全局尾端吸附（Global Tail Snapping）**： 無論區塊被拖曳到哪一條輸出軸的哪一個 X 軸位置，只要放開滑鼠，系統會自動判定該區塊在 1D 陣列中的順序，並在視覺上強制將其 X 軸座標對齊在『所有軸之中，最後一個區塊的右側』，確保時間軸上絕對沒有重疊或空白。
* **排版** :
    * 三條輸出軸是上下相連的。
    * 區塊或許位於不同的輸出軸上, 但所有區塊都共享同個時間軸。根據上述兩點, 區塊之間必須緊密相連、不可於時間軸上重疊。

## 互動與拖曳邏輯（Drag & Drop Interactions）

畫面上分為兩個主要區域：「側邊欄（模板庫）」與「主時間軸（實作區）」。

### １．側邊欄
* 分為兩個field : 預設區塊、自訂區塊。
    預設區塊的內容如同先前的說明, 是系統自帶的區塊；
    自訂區塊是使用者自訂的角色連招模板區塊, 三位角色的有各自的模板區塊。
* **實例化（Instantiation）**
    * **行為：** 從側邊欄拖出區塊至主軸。
    * **底層邏輯：** 側邊欄區塊屬性為「可拖曳（Draggable）但不參與主軸排序」。拖入主軸時觸發深拷貝（Deep Copy），產生一個帶有通用唯一辨識碼（UUID）的新實體（Instance）。

### ２．主時間軸
* 主時間軸中「同個輸出軸上」的區塊都能互相移動、複製貼上, 防止不同角色的連招混用。
* **自由新增**：使用者允許直接在主時間軸中新增空白的區塊，並自由編輯其連招文字。此類區塊在底層型別上同樣屬於 `InstanceBlock`，但其來源追蹤欄位 （originId） 將被記錄為 `null`。
* **序列化（Move & Serialization）**
    * **行為：** 將主時間軸內的任意區塊反向拖曳至側邊欄，將其新增為自訂的模板區塊。
    * **視覺概念：** 採「複製 (copy)」概念。拖曳完成後，原區塊依然留在主時間軸上。
    * **底層邏輯：** 複製該區塊的資料，賦予新的「模板 ID」後寫入側邊欄陣列。
    * 禁止從輸出軸序列化區塊到「非該名角色」的模板庫中。

### ３．刪除機制與快捷鍵
* **快捷刪除：** 支援鍵盤快捷鍵（Delete / Backspace）刪除。
* **拖曳刪除：** 當主時間軸的區塊被拖曳至畫面空白處（無效放置區）並放開時，直接從時間軸陣列中刪除（視為丟棄）。
* **其他快捷鍵**:
    * 多重選取(ctrl+左鍵 / 鼠標框選)
    * 剪下、複製、貼上、向右複製(ctrl+x/c/v/d)

### 4. 視覺效果與操作反饋
* **抓取浮空（Elevation & Scale）**:當滑鼠點擊並拖起區塊時，區塊微微放大，並在下方產生較深的陰影（Drop Shadow）；讓使用者明確知道目前操作哪個區塊。
* **選中提示**:當游標位於某個區塊上方時, 會有淡色的邊框醒目提示
* **刪除提示**拖曳區塊到無效放置區, 並且尚未鬆開左鍵時, 該區塊會呈現紅色條紋的警告樣式
* **平滑擠壓（Smooth Displacement）**：把區塊拖曳到主軸的兩個現有區塊之間時，後方的區塊不應該是「瞬間閃現」退後，而是要「平滑滑動（Transition）」讓出一個空位。
* **吸附歸位（Snapping）**：放開滑鼠時，區塊快速且帶有微小彈性（Spring Bounce）地吸附到最終位置。
* **預期落點提示（Drop Indicator）**：使用拖曳移動某個區塊時, 會顯示一條發光的「吸附線」, 表示要放置的位置(可能是區塊左/右側, 或兩個區塊之間; 如果該輸出軸為空, 就在輸出軸上呈現提示效果)。
* **模板建立成功提示（Toast Notification）**：當主軸區塊實例成功拖回側邊欄並轉為模板時，側邊欄的新區塊可以閃爍一次（Flash），或者在畫面角落彈出一個短暫的「已新增至模板庫」的小提示框。
* **複製殘影**:執行序列化（從主軸拖區塊回側邊欄）時，主軸上原有的區塊會保留一個半透明的『殘影』留在原地；而滑鼠游標上則是抓著一個實體的區塊實例, 直到鬆開滑鼠。

## 技術棧與工具建議（Tech Stack Recommendations）

* **開發環境：** VS Code + Git + GitHub
* **前端框架（Framework）：** React 或 Vue.js（搭配 TypeScript 進行型別檢查，確保資料結構穩定）。
* **拖曳套件（Drag & Drop Library）：** `dnd-kit`（React 生態系）或 `SortableJS`。處理底層物理拖曳、碰撞偵測與雙向區域規則。
* **狀態管理（State Management）：** `Zustand`（React）或 `Pinia`（Vue）。用於跨元件共享資料（如側邊欄與主軸之間的資料傳遞）。
* **資料持久化（Data Persistence）：** 瀏覽器的 `LocalStorage`。每次陣列更新時，將自訂模板資料同步存入本地端，防止重新整理網頁後資料遺失。
* **部署平台（Deployment）：** GitHub Pages 或 Vercel（免費自動化靜態網站代管）。
