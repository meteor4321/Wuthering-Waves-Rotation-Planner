# 架構說明

## 一、根目錄檔案（專案的基礎建設與設定）

- `.gitignore`
  - 告訴 Git 版本控制系統哪些檔案或資料夾不要上傳到 GitHub（例如肥大的套件包資料夾 `node_modules`）。

- `index.html`
  - 這是使用者打開網址時，瀏覽器最先讀取的唯一一個 HTML 檔案。
  - 它裡面通常只有一個空的 `<div>`，後續所有的 Vue 畫面都會動態塞進這個空殼裡（這就是 SPA 單頁面應用的原理）。

- `package.json`
  - 專案的身分證與物料清單。
  - 裡面記錄了專案名稱，以及安裝了哪些外部套件（如 Pinia、Tailwind、SortableJS）。

- `tsconfig.*.json`（包含 app 與 node）
  - TypeScript 的翻譯辭典設定檔。
  - 用來告訴編輯器要用多嚴格的標準來檢查程式碼，以及如何將 TypeScript 翻譯成瀏覽器看得懂的 JavaScript。

- `vite.config.ts`
  - Vite 是一款極速的打包工具。
  - 此檔案負責告訴 Vite 如何把無數個 `.vue` 與 `.ts` 檔案壓縮並打包成最終能上線的網頁檔案。

- `tailwind.config.ts`、`postcss.config.js`
  - Tailwind CSS（樣式工具）的設定檔。
  - 可以在這裡自訂鳴潮專屬的主題顏色（例如衍射黃、暗鳴紫），以便在程式碼中快速套用。

## 二、public/ 資料夾（靜態資源）

- 用途
  - 用來放置不需要被 Vite 打包壓縮的原始檔案。
  - 通常是全域共用的圖片或靜態資源。

- `favicon.ico`
  - 顯示在瀏覽器分頁標籤上的網頁小圖示。

## 三、src/ 資料夾（專案核心原始碼）

### 一、入口與大腦

- `main.ts`
  - 程式的第一個進入點。
  - 負責啟動 Vue 框架，並把各種外掛（如 Pinia 狀態管理）安裝上去。
  - 最後將畫面掛載到 `index.html`。

- `App.vue`
  - 專案的最外層容器。
  - 負責劃分螢幕的大版面（例如將畫面切成左側邊欄與右側主時間軸）。

### 二、資料與邏輯定義（抽象層）

- `types/` （型別定義）
  - 存放所有自訂的資料格式。
  - 例如 `block.ts` 定義了系統基礎區塊 （DefaultBlock）、側邊欄自訂模板 （TemplateBlock） 與時間軸實體 （InstanceBlock） 的泛用結構。

- `constants/`（常數定義）
  - 存放永遠不會變動的資料。
  - 例如鳴潮的角色名稱列表、預設的 7 種基礎招式。
  - 遊戲更新時只需修改單一檔案即可。

### 三、全域狀態與共用函式（運作中樞）

- `stores/`（Pinia 狀態管理）
  - 專案的中央資料庫。

  - `useRotationStore.ts`
    - 記錄目前時間軸上的資料陣列與排列狀態。

  - `useSidebarStore.ts`
    - 記錄側邊欄中有哪些使用者自訂模板。

  - 任何畫面元件（如區塊、垃圾桶）產生變動時，都會透過 Store 回報並更新資料。

- `composables/`（組合函式）
  - 將跨元件共用的行為邏輯抽離出來。
  - 例如拖曳功能背後涉及大量滑鼠座標計算，可集中寫在 `useBlockDrag.ts`。
  - 任何元件需要拖曳功能時，只需呼叫此函式即可，保持畫面程式碼整潔。

- `utils/`（工具函式）
  - 存放與畫面無關的純計算功能。

  - `uuid.ts`
    - 負責產生唯一識別碼。

  - `deepClone.ts`
    - 負責完整複製物件或陣列。

### 四、視覺與互動介面（畫面層）

- `components/`（Vue 元件）
  - 將複雜的網頁拆解成可重複使用的樂高積木。

  - `layout/`
    - 負責整體框架排版（如頂部導覽列）。

  - `sidebar/`
    - 側邊欄相關元件。
    - 包含預設區塊展示、自訂模板展示等功能。

  - `rotation/`
    - 主時間軸相關元件。
    - 包含泳道圖、落點吸附線等功能。

  - `character/`
    - 角色選擇下拉選單相關元件。

  - `ui/`
    - 最基礎的視覺元件。
    - 例如提示框（Tooltip）、帶有顏色標籤的區塊元件 `BlockChip.vue`。
