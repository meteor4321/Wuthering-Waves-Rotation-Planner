# SDD 與架構變更日誌（v1.1）

[刪除] 型別定義 (block.ts)：

- 變更說明：完全移除 BaseSkillKey 聯合型別（包含延奏技等），並且將 skills 陣列屬性從 BlockBase 中刪除。

- 原因：為了讓使用者能在主軸上自由新增空白區塊並輸入任意連招文字（甚至亂碼）。區塊邏輯正式與「嚴格基礎招式綁定」解耦，改為純粹依賴 label 文字做視覺寬度與內容的呈現。

[修改] 區塊結構泛用化 (block.ts)：

- 變更說明：不再區分 instanceId 或 templateId，所有區塊（DefaultBlock、TemplateBlock、InstanceBlock）統一使用泛用的 id: string 作為唯一識別碼。

- 變更說明：InstanceBlock 的 originId 欄位型別修改為 string | null。當使用者在主軸上直接新增空白/自訂區塊時，該區塊的 originId 即為 null，不再需要額外建立 Freeform 類型的區塊。

[修改] SDD文件更新 -> 核心機制與視覺設計 -> 1. 區塊 (Block)：

- 新增概念：使用者現在允許於主時間軸內「直接新增空白實體區塊」並自由編輯文字，且「不強制要求」使用者必須將其序列化儲存為模板。
